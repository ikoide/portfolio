Creating a New Hostname on Every System Reboot

Posted by Ian Koide on April 10th, 2020

If you're looking for a way to create a new hostname for your system on every reboot you've come to the right place. The program can be found here. The setup is minimal and requires only the cloning of a github repository and the creation of a cron job.

$ git clone https://github.com/ikoide/HostnameReboot

After cloning the repository you have to create a cron job on the root user.

$ sudo crontab -e

Inside the crontab file you have to append this line to the file. You may remove the commented lines if you'd like.

$ @reboot /path/to/HostnameReboot/new_hostname.sh

For example I'd use the following line.

$ @reboot /home/lan/HostnameReboot/new_hostname.sh

One last thing, you have to edit the file named config in the repo directory. Change the line in config to the path to your wordlist.

Now all you've got to do is reboot your system. You can open a new terminal session and see that your hostname has been updated to a random english word. If you'd like to use your own words for new hostnames read on.

(humanenesss?)

In order to change the words that the program selects from, edit the words.txt file.
